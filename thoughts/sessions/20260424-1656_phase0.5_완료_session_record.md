## 🎓 배운 점 / 주의사항

1. **Supabase Storage RLS**: SQL Editor 에서 `storage.objects` 에 POLICY CREATE/DROP 불가 (Supabase 2024~2025 정책). Dashboard UI 로만 설정.
2. **Server Action 변환**: `'use server'` 선언만으로 firebase-admin 등 서버 전용 패키지를 client 번들에서 격리 가능.
3. **React 19 `react-hooks/set-state-in-effect`**: 데이터 로딩 패턴이 대량 flagged. 마이크로태스크 지연 `Promise.resolve().then(fn)` 로 회피 가능.
4. **Vercel serverless timeout**: Server Action 전환 시 HTTP 왕복 오버헤드로 기존 parallel 호출이 timeout 초과 가능. Chunk 배치 필수.
5. **브랜치 보호 `required_status_checks`**: 실제 check 이름과 정확히 일치해야 함. `pr-check` 같은 가상 이름 설정 시 영구 차단.
6. **한국어 파일명**: Supabase Dashboard UI 가 거부 (service_role 스크립트로 우회 필요).
