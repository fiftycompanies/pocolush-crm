### 현 상태
- `lib/supabase/client.ts`: singleton + `accessTokenFn` 옵션 적용 (커밋 9de7b51)
- 그러나 prod에서 여전히 WebSocket 401 socketerror 발생
- TopBar setAuth (5361889) + use-bbq-board setAuth (이전) 적용 + 본 fix 추가 → 모두 동작 안 함

### 의심 원인
```
_client = _make(url, key);   // ① createBrowserClient 호출 (동기)
                              //   ↓ 내부에서 realtime client 즉시 초기화
                              //   ↓ accessTokenFn 호출 — but _client는 아직 null !
                              //   ↓ null 반환 → anon token 사용 → 401
                              // ② 함수 반환 후 _client 변수에 할당 (이미 늦음)
```

### 해결 옵션 (다음 세션 검토)
1. **명시적 setAuth 즉시 호출** (가장 빠름):
   ```ts
   _client = _make(url, key);
   (async () => {
     const { data: { session } } = await _client.auth.getSession();
     if (session?.access_token) _client.realtime.setAuth(session.access_token);
   })();
   ```
2. **localStorage 동기 읽기**:
   ```ts
   const supabaseKey = `sb-${projectRef}-auth-token`;
   const stored = typeof window !== 'undefined' ? localStorage.getItem(supabaseKey) : null;
   const initialToken = stored ? JSON.parse(stored).access_token : null;
   // initialToken을 옵션에 전달
   ```
3. **supabase-js createClient 직접 호출** (ssr wrapper 우회):
   - @supabase/supabase-js createClient 사용
   - SSR cookie 동기화는 별도 처리
4. **Vercel logs + supabase-js debug 모드** 활성화하여 connection 흐름 trace
